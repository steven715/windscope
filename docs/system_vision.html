一句話總結(v2 跟 v1 的差別)

不再只是「整理事實給你看」——系統每天 08:50 自己下判斷(偏多/偏空/中性 + 理由 + 信心度), 13:30 收盤後自己對答案,把命中與否記錄下來。長期累積後,你不只看每天的判斷, 還看得到這套判斷邏輯歷史上到底準不準。整套以常駐 server 運行:排程內建、有 Web 畫面、原始/衍生/判斷資料都能查。

從現在到 v2 的路徑

✅ 已完成(全部沿用,一行不浪費)

  • 4 個 collectors + 統一 HTTP client
  • 3 組衍生指標計算
  • SQLite schema + 冪等寫入 + backfill
  • 文字摘要產生器(變成通知的內容來源)
  • 155 個測試

🔨 v2 新增的工作

  • Server 主體:常駐進程 + 內建排程器(取代 crontab)
  • Layer 3 訊號引擎:規則式判斷 + 理由記錄(signals 表)
  • Layer 4 驗證引擎:收盤後對答案 + 命中率統計(verifications 表)
  • Web 畫面:dashboard + 四類資料查詢頁
  • 通知模組(原 Round 5 範圍)

🚫 仍然不做的

  • 下單 / 任何交易執行
  • 盤中即時監控(僅開盤前 + 收盤後兩個時點)
  • 回測引擎(驗證是「前瞻實測」,不是歷史回測)
  • 機器學習模型(先用透明規則累積戰績)

待決定的事:① 訊號規則的具體門檻(例如幾個維度同向才算偏多)② Web 技術選型(傾向 FastAPI + APScheduler + 簡單 server-rendered 頁面,單機即可)③ 驗證的「實際走勢」用什麼衡量(加權指數收盤漲跌?開盤跳空方向?兩者都記?)④ server 跑在哪台機器(需要一台常開的)。
